# Code Review Checklist

## 业务正确性
- [ ] 变更是否覆盖主流程 + 异常流程？
- [ ] 边界输入（空、0、负数、超长、非法枚举）是否处理？
- [ ] 是否破坏向后兼容（接口字段、序列化、DB schema）？

## 数据与事务
- [ ] 事务边界正确？异常是否回滚？
- [ ] 幂等是否保障（重试、重复消息、重复提交）？
- [ ] 缓存与DB一致性：先写DB再删缓存？是否有双写风险？

## 安全
- [ ] 鉴权/权限校验齐全？
- [ ] 注入风险（SQL/命令/路径/模板）？
- [ ] 日志/返回中是否包含敏感信息？

## 性能与稳定性
- [ ] 是否引入N+1/无界分页/全表扫描？
- [ ] 网络调用是否设置超时、重试、熔断？
- [ ] 资源是否泄露（文件句柄/连接/线程）？

## 可维护性
- [ ] 命名清晰、职责单一、复杂度可控？
- [ ] 重复逻辑是否抽取？
- [ ] 日志可观测性：关键字段、traceId、错误码？
